The rules of the single table design

  1. Have attribute TYPE in all the rows. It defines what kind of row is in question.
  2. Have attribute id in all the rows.
  3. Name keys with uppercase, preferably PK, SK, GSI1PK, GSI1SK, ...
  4. Use only global secondary indices, not local. Project all attributes to global secondary indices.
  5. Keys consist of # separated strings (and are therefore always of type string). You can use in the keys UPPERCASE constants and any simple attribute (string, number, boolean) that exists in the row or it's lowercase or uppercase variant.
  6. Study DynamoDB before you start using it.
